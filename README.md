# SputnikAuthV2

Система идентификации спутников по угловым наблюдениям с использованием TLE-каталогов Celestrak и модели SGP4.

Проект решает задачу: по серии наблюдений вида `время + азимут + высота над горизонтом` определить, какой спутник из выбранного каталога мог породить наблюдаемую траекторию. Поддерживается open-set постановка: наблюдаемый объект может отсутствовать в выбранном каталоге.

# Сборка проекта (Windows + MinGW + vcpkg)

## Требования

* CMake >= 3.16
* Ninja
* MinGW / Strawberry
* vcpkg
* Python 3: для построения экспериментальных графиков

---

## Установка зависимостей

```powershell
cd vcpkg
.\bootstrap-vcpkg.bat
.\vcpkg.exe install curl:x64-mingw-dynamic
```

---

## Сборка проекта

Из корня проекта:

```powershell
Remove-Item -Recurse -Force build
mkdir build
cd build

cmake .. -G Ninja -DCMAKE_BUILD_TYPE=Release -DCMAKE_TOOLCHAIN_FILE="../vcpkg/scripts/buildsystems/vcpkg.cmake" -DVCPKG_TARGET_TRIPLET=x64-mingw-dynamic
cmake --build .
```

---

## Запуск

Из папки `build`:

```powershell
.\SputnikAuthV2.exe
```

При запуске программа:

* предлагает обновить TLE-каталоги;
* предлагает выбрать один или несколько каталогов для анализа;
* генерирует синтетические наблюдения видимого прохода;
* запускает идентификацию спутника;
* выводит top-кандидатов, `chi-square`, вероятность внутри каталога и `p-value`;
* при необходимости запускает экспериментальный модуль.

# Загрузка каталогов Celestrak

В проекте реализована загрузка TLE напрямую с Celestrak:

```text
https://celestrak.org/NORAD/elements/gp.php?GROUP=<group>&FORMAT=TLE
```

Загрузка выполняется через `libcurl`, после чего данные сохраняются в локальный кэш:

```text
data/<category_slug>/<group_slug>.tle
```

Примеры групп:

* `stations`
* `starlink`
* `active`
* `weather`
* `gps-ops`
* `gnss`

Поддерживается выбор нескольких каталогов и объединение их в один каталог кандидатов. Дубликаты удаляются по `NORAD ID`.

# Архитектура

Основной ход работы программы помодульно:

```text
Console -> Data -> Observation -> Prepare -> Core -> Output -> Experiments
```

Модули:

* `ConsoleInterface` — ввод пользователя, выбор каталогов, запуск экспериментов.
* `DataModule` — загрузка TLE из файлов и из интернета.
* `TLEParser` — парсинг TLE-записей.
* `ObservationGenerator` — генерация синтетических наблюдений для самопроверки.
* `PrepareModule` — предобработка и проверка входных данных.
* `CoreModule` — основной алгоритм идентификации.
* `OutputModule` — вывод результатов.
* `ExperimentRunner` — автоматические эксперименты и CSV для графиков.

# Алгоритм идентификации

Для каждого спутника-кандидата:

1. TLE преобразуется в объект `libsgp4::Tle`.
2. SGP4 распространяет орбиту на моменты наблюдений.
3. Положение спутника переводится в топоцентрические углы наблюдателя: азимут и высоту.
4. Расчетные углы сравниваются с измеренными.
5. Дополнительно учитывается угловая скорость между соседними наблюдениями.
6. Считается статистика `chi-square`.
7. Кандидаты ранжируются по возрастанию `chi-square`.
8. Для лучшего кандидата считается `p-value`, чтобы проверить, может ли объект отсутствовать в каталоге.

# Open-set проверка

В проекте используются две гипотезы:

```text
H_in  - наблюдаемый объект есть в выбранном каталоге
H_out - наблюдаемый объект отсутствует в выбранном каталоге
```
Если правильного спутника нет в каталоге, кто-то все равно будет лучшим. Поэтому дополнительно используется `p-value` критерия согласия:

```text
p-value = P(ChiSquare(df) >= observed_chi2)
```

Если `p-value < alpha`, лучший кандидат плохо объясняет наблюдения, и программа сообщает, что объект, вероятно, отсутствует в каталоге.

Текущий порог:

```text
alpha = 1e-3
```

# Эксперименты

В проекте реализованы эксперименты:

* устойчивость к угловому шуму;
* каталог похожих ложных кандидатов;
* open-set проверка, когда target удаляется из каталога;
* проверка разных типов орбит: LEO, MEO, GEO, Polar, Elliptical;
* измерение времени работы при разном размере каталога.

Результаты сохраняются в:

```text
experiments/results/
```

Графики сохраняются в:

```text
experiments/figures/run_<timestamp>/
```

Построение графиков:

```powershell
python experiments\plots\plot_experiments.py
```
